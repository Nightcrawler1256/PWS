1. <h1> - <h6>
    • Used to define headings in a document, with <h1> being the highest level and <h6> the lowest.
    • Best practice is to use only one <h1> per page for the main heading, followed by hierarchical usage of the others for subheadings.
    • Heading tags should convey the semantic structure of the content, not just the visual appearance.
  2. <p>
    • Defines a paragraph of text.
    • Best practice is to use <p> for separate blocks of text to ensure proper spacing and semantic meaning.
    • Avoid using multiple <p> tags for single lines of text; instead, use CSS for styling.
  3. <em>
    • Indicates emphasis, typically rendered as italic text by default.
    • Best practice is to use <em> for semantic emphasis, not just for visual styling purposes.
    • Should not be used solely for styling; use CSS for that purpose.
  4. <strong>
    • Indicates strong importance, typically rendered as bold text by default.
    • Best practice is to use <strong> for semantic importance, not just for visual styling purposes.
    • Should not be used solely for styling; use CSS for that purpose.
  5. <a>
    • Defines a hyperlink, linking to another webpage or resource.
    • Best practice is to use the <a> tag with the href attribute to create clickable links.
    • Ensure links have descriptive text for accessibility and SEO purposes.
